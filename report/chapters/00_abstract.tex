Quantum algorithms such as Grover's search and the swap test suggest possible
speed-ups for vector search, the operation that retrieves stored items whose
embeddings are closest to a query, but their practical value at present-day
scale is unclear. We built a benchmark framework in which seven classical,
quantum, and hybrid engines answer the same queries over the same CLIP
embeddings, evaluated on a fixed twenty-image cross-modal dataset across
embedding dimensions and measurement shot counts. Accuracy was measured with
Mean Reciprocal Rank and cost with comparisons, oracle calls, qubits, and
circuit depth; the quantum engines ran mainly on a noiseless Qiskit simulator,
with one small IBM hardware run to confirm the execution path. The results show
no practical quantum advantage for this task. Accuracy followed the embedding
dimension rather than whether an engine was quantum: the swap test matched the
classical ranking at dimension 256, and Grover matched its expected oracle-call
count, yet no quantum search engine beat the classical graph-search baseline.
Grover's apparent speed-up counts only the search after the data is loaded into
the oracle, so building that oracle restores the main retrieval cost. We conclude
that cross-modal vector search is mainly a data-access problem rather than one
where quantum computation is the bottleneck, under current methods and hardware.

\vspace{1em}
\noindent\textbf{Keywords:} \emph{quantum computing, vector search, similarity search, swap test, Grover's algorithm, CLIP, benchmarking}
