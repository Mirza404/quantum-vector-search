Vector search finds stored items whose numerical representations, called
embeddings, are closest to a query. It is used in systems such as text-to-image
retrieval, where a caption is used to find matching images. Quantum algorithms
such as Grover's search and the swap test suggest possible speed-ups for this
kind of similarity search, but their practical value depends on data loading,
hardware noise, and comparison with strong classical indexes.

We built a benchmark framework in which seven engines answer the same queries
over the same CLIP (Contrastive Language-Image Pre-training) embeddings. The
engines include exact classical search, approximate graph search, quantum
similarity and search variants, and a hybrid graph-search plus quantum-reranking
method. We tested them on a fixed twenty-image dataset across embedding
dimensions and measurement shot counts. Accuracy was measured with Mean
Reciprocal Rank, while cost was measured with comparisons, oracle calls, qubits,
and circuit depth. The quantum engines ran mainly on a noiseless Qiskit
simulator; one small IBM hardware run was used only to check that the execution
path works on a real quantum processor.

The measured results do not show a practical quantum advantage for this task.
Accuracy mainly followed embedding dimension, not whether the engine was quantum.
The swap test matched the classical ranking at dimension 256 when enough
measurements were used, and Grover matched its expected oracle-call count at the
tested size. However, no quantum search engine beat the classical graph-search
baseline. Grover's apparent speed-up counts only the search after the data is
already available to the oracle; building that oracle or loading the data would
add back the main retrieval cost. We conclude that cross-modal vector search is
mainly a data-access problem, not a task where quantum computation is the
bottleneck, under current methods and hardware.
